## 5. Proof / disproof / reduction

### A. Lifted zero catalog

For the first four slots, the exact lifted differences are, up to sign:

```text
d_1,
h,
h+d_1,
h-d_1.
```

Since `d_1` lies in the dyadic nonzero shell, the lifted equation `d_1=0` is
excluded by the model range. The remaining internal first-block exact-zero
diagonals are:

```text
h=0,
h=-d_1,
h=d_1.
```

For the second block, similarly:

```text
k=0,
k=-d_2,
k=d_2.
```

For cross-block differences, set `r=x-y`. The exact lifted equations are:

```text
r = -a d_1 - b h + c d_2 + e k,
```

where:

```text
(a,b),(c,e) in {(1,0),(0,0),(1,1),(0,1)}.
```

There are sixteen cross-block lifted zero families.

Classification:

```text
LiftedZeroCatalog_339:
  STRUCTURAL / EXTRACTION.

InternalZeroDiagonalCatalog_339:
  STRUCTURAL / EXTRACTION.

CrossZeroDiagonalCatalog_339:
  STRUCTURAL / EXTRACTION.
```

This is a catalog only. It does not estimate the weighted row.

### B. Relation to Module 324-325 structural diagonals

The lifted zero families are the integer-lift version of the Module 324
structural diagonal catalog:

```text
Z_struct^8 = union_{i<j} { Delta_{ij}=0 in G_N }.
```

The distinction matters. A cyclic zero may occur through wraparound, while a
lifted zero depends on the declared representatives. Conversely, if a lifted
zero occurs then the corresponding cyclic zero occurs.

The exact-zero tail row is therefore not a new shape of diagonal, but it is a
stronger weighted demand:

```text
structural diagonal
  + low-prime cover envelope
  + high-prime zero-tail contribution
  + absolute localized kernel
  + threshold masks.
```

Classification:

```text
StructuralDiagonalTransferCriterion_339:
  CONDITIONAL.
```

If `StructuralDiagonalRows_325` were proved with this same low-prime envelope,
zero-tail weight, absolute kernel, masks, dyadic shell, W-residue convention,
and limit order, then it could feed the exact-zero tail row. The existing
catalog does not supply that theorem.

### D. Internal diagonal rows

The internal lifted zeros are:

```text
h=0, h=d_1, h=-d_1,
k=0, k=d_2, k=-d_2.
```

These rows interact directly with:

```text
t=h+k
```

inside `KAbs_{U,V}^0(d_1,d_2;t)`. For example, on `h=0`, the kernel is
evaluated at `t=k`; on `h=-d_1`, at `t=k-d_1`.

Classification:

```text
InternalZeroKernelRows_339:
  OPEN.
```

The zero-mean identities of the signed minor kernel do not apply after taking
absolute values and multiplying by the low-prime envelope.

### E. Cross diagonal rows

The cross lifted zeros are:

```text
x-y = -a d_1 - b h + c d_2 + e k.
```

For an unweighted average over `x,y`, each such equation is codimension one.
But the exact row is not unweighted: it contains residual products, the
low-prime envelope, high-prime zero-tail contributions, masks, and the
absolute localized kernel.

Classification:

```text
CrossZeroKernelRows_339:
  OPEN.
```

Codimension one is only a counting diagnostic unless transferred to the
actual weighted row.

## 6. Edge cases and exclusions

- Exact lifted zero is not controlled by `sum_{p>Y,p|A}1/p` for nonzero `A`.
- A cyclic zero may be a wraparound event rather than a lifted zero.
- Internal diagonals interact with the kernel through `t=h+k`.
- Cross diagonals are codimension one only before inserting residual weights,
  low-prime envelopes, high-prime zero-tail contributions, masks, and the
  absolute kernel.
- The off-diagonal condition `d_1 != d_2` does not remove cross-block
  diagonals.
- The dyadic nonzero shell excludes `d_i=0` as an integer, but not all
  internal equations involving `h` or `k`.
- Signed zero-mean cancellation is not available after absolute values.
- The model selector is not the actual sharp moving selector.
- The local W-cyclic prime-only model is not the original full-gap problem.
- A diagonal catalog is not an analytic proof.
